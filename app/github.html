Git & GitHub
desde VSCode

Conecta tu proyecto local con un repositorio remoto en GitHub usando la terminal integrada de Visual Studio Code.

Terminal Git GitHub

Antes de empezar

Requisitos previos

Visual Studio Code

Versión estable instalada en tu equipo.

Git instalado

Verifica con git --version en la terminal.

Cuenta en GitHub

Sesión activa en github.com.

1

Paso 1

Abre la terminal en VSCode

La terminal integrada inicia directamente en la raíz de tu proyecto.

Atajo de teclado

Ctrl + ` en Windows/Linux
⌘ + ` en macOS

Menú

Ve a Ver → Terminal en la barra de menú.

2

Paso 2

Inicializa el repositorio local

Convierte tu carpeta en un repositorio Git, creando la carpeta oculta .git.

$ git init
# Initialized empty Git repository in /tu/proyecto/.git/
3

Paso 3

Configura tu identidad en Git

Git necesita saber quién hace cada commit. Esta configuración es global.

# Reemplaza con tus datos reales
$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tu@correo.com"

# Verifica que quedó bien
$ git config --list
4

Paso 4

Crea el repositorio en GitHub

Necesitas un repositorio remoto vacío en GitHub.

  1. a

    Ve a github.com/new.

  2. b

    Escribe el nombre y elige público o privado.

  3. c

    No marques README, .gitignore ni licencia — debe quedar vacío.

  4. d

    Copia la URL que aparece al crear el repositorio.

# La URL tendrá este formato:
https://github.com/tu-usuario/nombre-repositorio.git
5

Paso 5

Conecta el repositorio remoto

Le dices a tu Git local dónde está el repositorio remoto en GitHub.

# Agrega el remoto (reemplaza la URL)
$ git remote add origin https://github.com/tu-usuario/nombre-repositorio.git

# Verifica que el remoto quedó registrado
$ git remote -v
# origin  https://github.com/tu-usuario/nombre-repositorio.git (fetch)
# origin  https://github.com/tu-usuario/nombre-repositorio.git (push)
6

Paso 6

Prepara y sube tu primer commit

Añade archivos al staging, crea el commit, renombra la rama y sube el código.

# 1. Agrega todos los archivos al staging
$ git add .

# 2. Revisa qué archivos están listos (opcional)
$ git status

# 3. Crea el commit
$ git commit -m "feat: primer commit del proyecto"

# 4. Renombra la rama local a 'main'
$ git branch -M main

# 5. Sube y vincula la rama local con la remota
$ git push -u origin main

Referencia rápida

Flujo de trabajo diario

Después del primer push, este será tu ciclo habitual.

# Baja cambios del remoto
$ git pull

# Trabaja en tus archivos...

# Revisa qué cambió
$ git status
$ git diff
# Agrega al staging
$ git add .

# Crea el commit
$ git commit -m "descripción del cambio"

# Sube al remoto
$ git push

Solución de problemas

Errores más comunes

Ya existe un remoto con el nombre origin. Tienes dos opciones: 
# Opción A: elimina y vuelve a agregar
$ git remote remove origin
$ git remote add origin https://github.com/tu-usuario/repo.git

# Opción B: actualiza la URL directamente
$ git remote set-url origin https://github.com/tu-usuario/repo.git

El remoto tiene commits que no tienes localmente. Integra primero esos cambios: 
$ git pull --rebase origin main
# Resuelve conflictos si los hay, luego:
$ git push

Desde 2021 GitHub no acepta contraseñas por HTTPS. Usa un token de acceso personal (PAT):
  1. Ve a GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic).
  2. Genera un token con permisos repo.
  3. Úsalo como contraseña cuando Git te la pida.

No hay cambios nuevos para commitear. Guarda cambios en algún archivo y vuelve a ejecutar git add . y git commit.
