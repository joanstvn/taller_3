Ingresar

Guía práctica

Git & GitHub
desde VSCode

Conecta tu proyecto local con un repositorio remoto en GitHub usando la terminal integrada de Visual Studio Code.

Terminal Git GitHub 
# Flujo completo en 6 pasos
$ git init
$ git add .
$ git commit -m "init"
$ git remote add origin
$ git branch -M main
$ git push -u origin main

En esta guía

Requisitos previos 1. Abre la terminal 2. Inicializa Git 3. Configura tu identidad 4. Crea el repositorio 5. Conecta el remoto 6. Tu primer push Flujo diario Errores comunes

Antes de empezar

Requisitos previos

Visual Studio Code

Versión estable instalada en tu equipo.

Git instalado

Verifica con git --version en la terminal.

Cuenta en GitHub

Sesión activa en github.com.

1

Paso 1

Abre la terminal en VSCode

La terminal integrada de VSCode inicia directamente en la raíz de tu proyecto abierto.

Atajo de teclado

Presiona Ctrl + ` en Windows/Linux
o ⌘ + ` en macOS.

Menú

Ve a Ver → Terminal en la barra de menú.

Asegúrate de haber abierto la carpeta raíz de tu proyecto en VSCode antes de abrir la terminal.
2

Paso 2

Inicializa el repositorio local

Este comando convierte tu carpeta en un repositorio Git local, creando la carpeta oculta .git.

$ git init
# Initialized empty Git repository in /tu/proyecto/.git/
Si ya clonaste el repositorio con git clone, omite este paso. El repositorio ya viene inicializado.
3

Paso 3

Configura tu identidad en Git

Git necesita saber quién hace cada commit. Esta configuración es global.

# Reemplaza con tus datos reales
$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tu@correo.com"

# Verifica que quedó bien
$ git config --list
Usa --local en lugar de --global si necesitas una identidad distinta solo para este proyecto.
4

Paso 4

Crea el repositorio en GitHub

Necesitas un repositorio remoto vacío en GitHub.

  1. a

    Ve a github.com/new.

  2. b

    Escribe el nombre y elige público o privado.

  3. c

    No marques README, .gitignore ni licencia. Debe quedar vacío.

  4. d

    Copia la URL que aparece al crear el repositorio.

# La URL tendrá este formato:
https://github.com/tu-usuario/nombre-repositorio.git
5

Paso 5

Conecta el repositorio remoto

Le dices a tu Git local dónde está el repositorio remoto en GitHub.

# Agrega el remoto (reemplaza la URL)
$ git remote add origin https://github.com/tu-usuario/nombre-repositorio.git

# Verifica que el remoto quedó registrado
$ git remote -v
# origin  https://github.com/tu-usuario/nombre-repositorio.git (fetch)
# origin  https://github.com/tu-usuario/nombre-repositorio.git (push)
SSH vs HTTPS: Con HTTPS GitHub pedirá usuario y token. Con SSH solo configuras tu clave una vez. Para proyectos frecuentes, SSH es más cómodo.
6

Paso 6

Prepara y sube tu primer commit

Añade tus archivos al staging, crea el commit, renombra la rama y sube el código.

# 1. Agrega todos los archivos al staging
$ git add .

# 2. Revisa qué archivos están listos (opcional)
$ git status

# 3. Crea el commit con un mensaje descriptivo
$ git commit -m "feat: primer commit del proyecto"

# 4. Renombra la rama local a 'main'
$ git branch -M main

# 5. Sube el código y vincula rama local con la remota
$ git push -u origin main
¿Qué hace el flag -u? Vincula tu rama local main con origin/main. Luego puedes usar solo git push y git pull.

Referencia rápida

Flujo de trabajo diario

Después del primer push, este será tu ciclo de trabajo habitual.

# Antes de empezar — baja cambios del remoto
$ git pull

# Trabaja en tus archivos...

# Revisa qué cambió
$ git status
$ git diff
# Agrega los cambios al staging
$ git add .

# Crea el commit
$ git commit -m "descripción del cambio"

# Sube al remoto
$ git push

Solución de problemas

Errores más comunes

Ya existe un remoto con el nombre origin. Tienes dos opciones: 
# Opción A: elimina y vuelve a agregar
$ git remote remove origin
$ git remote add origin https://github.com/tu-usuario/repo.git

# Opción B: actualiza la URL directamente
$ git remote set-url origin https://github.com/tu-usuario/repo.git

El remoto tiene commits que no tienes localmente. Primero integra esos cambios: 
$ git pull --rebase origin main
# Resuelve conflictos si los hay, luego:
$ git push

Desde 2021 GitHub no acepta contraseñas por HTTPS. Debes usar un token de acceso personal (PAT) como contraseña:
  1. Ve a GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic).
  2. Genera un nuevo token con permisos repo.
  3. Úsalo como contraseña cuando Git te la pida.

No hay cambios nuevos para commitear. Guarda cambios en algún archivo y vuelve a ejecutar git add . y git commit.
